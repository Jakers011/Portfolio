What makes this page work?

Normally I make sites that have a multitud of webpages, so this was my first attempt at making a single-page website. For basic libraries, I utilized bootstrap for the top navigation bar and for responsiveness. I then used a jquery function along with animate in order to create the scrolling effect. The scrolling effect should be a constant speed no matter from and to where you are going to on the webpage.

For pre-defined js libraries, I used the following libraries. First, a combination of Masonry and Isotope in order to create my Portfolio section. Masonry is pretty much used as a display option for Isotope in this scenario, where I just change the CSS of the cells to get it to look correct. Isotope needed to be initialized through jQuery, and then utlilizes jQuery on a button press in order to filter out selections. Its much easier frankly than me having to go and create a jQuery function manually that adds a hidden or deletes that css. This will be used more in depth at a later time, specifically when I add my personal websites as well as other class's assignments to the mosaic.

The blog utilizes a carousel javascript library called Flickity which creates responsive carousels. Flickity is initialized through a simple jQuery command. It operates by utilizing a preset css and js package, in which js functions can be changed as well as of course overwriting the css. Specifically, I designate a container dom which will be where the carousel operates and the library will reference, and then container cells within that dom that the library manipulates.

Final Project Blog Ideas

My idea is one of the following: 1. make a SAT quiz app that caters towards the user (I have worked on this for a company, however it was scrapped and now I will probably be re-hired to be lead developer for this project). or 2 and the most likely option: a scheduling app for those tenting in KVille. For both the basic components will be Vue and Firebase (Vue for connecting to Firebase and Firebase itself maybe for logging in but definitely for database storage). For the scehduling app (thats going to be the proposal from here on out). It needs to be a collaborative process (for up to twelve people) in which there will be a designated leader that can invite other tent members to sign up for their specific tent... but each person has their own schedule. Each tent member will fill out their availability by lets say just filling in buttons corresponding to times on certain days, and will do that on a weekly basis. However, I want the timing to be able to be variable, as some like 15 minute increments on determining availability and some like lets say an hour. Then everyones availability will be shown to the tent captain, maybe like in a google spreadsheet. The captain will assign a person to a time, which will both increment that person's total hours as well as the number of people present at that particular timing. Once the schedule has been completely filled out, it can then be published to a Calander displayed on the website to all members of the grouop, either made by myself or using Google Calander. I would also like to add a notification feature that will send alerts or an email to the individual alerting them when their scheduled slot is approaching... and the alerts need to be customizable as well.
